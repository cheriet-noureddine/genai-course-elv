# الذكاء الاصطناعي في حياتي اليومية — نسخة الخادم/الواجهة (Backend + Frontend)

هذا المشروع يستبدل ملف HTML الواحد الذي كان يحتوي على الدورة كاملة (كل الدروس
والإجابات الصحيحة مضمّنة في كود JavaScript يمكن لأي زائر فتحه من "عرض المصدر")
بتطبيق حقيقي مكوّن من:

```
project/
├── server/            ← خادم Node.js/Express (يحمل كل البيانات الحساسة)
│   ├── server.js
│   ├── package.json
│   └── data/
│       ├── structure.json     ← القائمة/الفهرس فقط (عناوين، أوقات) — بدون نصوص الدروس أو الإجابات
│       ├── answer-key.json    ← الإجابات الصحيحة + كلمة المرور تُقرأ من env — لا تُرسل للمتصفح كاملة أبداً
│       └── lessons/*.json     ← كل درس في ملف مستقل، يُرسل واحداً فواحد فقط عند الطلب
└── public/             ← الواجهة (تُخدَّم كملفات ثابتة)
    ├── index.html
    └── app.js          ← يتصل بالـ API بدلاً من احتواء الدورة كاملة
```

## ما الذي تغيّر عن النسخة الأصلية؟

1. **المحتوى انتقل إلى الخادم.** كل الدروس أصبحت ملفات JSON مستقلة تحت
   `server/data/lessons/`، بدلاً من أن تكون مكتوبة داخل كود الصفحة.
2. **الواجهة تطلب درساً واحداً فقط في كل مرة** عبر
   `GET /api/lesson/:mi/:li` — لا يوجد أي طلب يُرجع الدورة كاملة.
3. **الإجابات الصحيحة والشروحات لا تُرسل مسبقاً.** أسئلة الاختبارات تُرسل
   بدون حقل `correct` أو `explain`؛ التصحيح يحدث على الخادم عبر
   `POST /api/quiz/:mi/submit` و `POST /api/lesson-quiz/:mi/:li/submit`،
   وتُعاد النتيجة والشرح فقط بعد الإرسال (كما هو متوقع في أي اختبار).
4. **كلمة المرور تُتحقق منها على الخادم فقط** (`POST /api/login`)، ولم تعد
   موجودة كنص صريح في كود الواجهة. يمكن ضبطها عبر متغيّر البيئة
   `COURSE_PASSWORD` (القيمة الافتراضية إن لم تُحدَّد: `selouani-school`).
5. **حقل كلمة المرور يبقى غير مموّه (`type="text"`)** كما طُلب — بدون نقاط
   إخفاء الأحرف.
6. حماية أساسية أُضيفت لتجنّب المشاكل الشائعة: تحقق من صلاحية معرّفات
   الوحدة/الدرس في الروابط (لمنع محاولات الوصول لملفات خارج مجلد البيانات)،
   وتحديد عدد محاولات كلمة المرور الخاطئة، ومعالجة أخطاء الاتصال في الواجهة.

## التشغيل محلياً

```bash
cd server
npm install
# اختياري: لتغيير كلمة المرور بدلاً من القيمة الافتراضية
export COURSE_PASSWORD="كلمة-مرور-جديدة"
npm start
```

ثم افتح المتصفح على `http://localhost:3000`.

كلمة المرور الافتراضية (إن لم تُغيَّر): `selouani-school`

## ملاحظات حول تقدّم الطالب (Progress)

تقدّم الطالب (الدروس المكتملة، والاختبارات المُنجزة) يبقى محفوظاً في متصفح
الطالب فقط (`localStorage`) — لا يحتاج قاعدة بيانات على الخادم لأنه لا يحتوي
على معلومات حساسة. يمكن نقله إلى الخادم مستقبلاً بسهولة (مثلاً في ملف/قاعدة
بيانات لكل طالب) إن احتاج المعلم لمتابعة تقدّم الطلاب مركزياً.

## نشر المشروع (Deployment)

هذا خادم Node.js عادي (Express) وليس ملف HTML ثابتاً، فيحتاج بيئة تستطيع
تشغيل `node`، مثل: Render، Railway، Fly.io، أو أي خادم VPS. لا يعمل على
استضافة ثابتة (Static Hosting) فقط مثل GitHub Pages.
